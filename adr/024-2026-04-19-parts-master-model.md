# ADR 024: Parts Master Model and Migration

## Metadata
- Date: 2026-04-19
- Status: Accepted
- Authors: The AI
- Drivers: F-025 requirement for parts master data
- Tags: model, migration, SQLAlchemy, Alembic

## Decision
Implement a `Part` model and Alembic migration for the `part` table, supporting master parts records and CRUD operations.

## Context
The system requires a master parts record for cataloging and linking to SAP data. This supports CRUD, search, and integration with Access forms.

## Alternatives Considered
- No dedicated model: Would prevent structured part management.
- Manual SQL: Increases maintenance and error risk.

## Implementation Details
- SQLAlchemy model: `Part` with fields `part_id`, `part_number`, `description`, `active`.
- Alembic migration: Autogenerated for table creation.
- Unit test: Validates model fields.

## Validation
- All tests pass (pytest)
- Migration applies cleanly

## Consequences
- Enables structured part management
- Supports future CRUD and integration
